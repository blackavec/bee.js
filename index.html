Bee.js | Ajax framework

Bee.js is frontend javascript's framework which made to help developers to create the better websites.
ajax is the concentration point on this project.

Welcome to Bee.js Pages

long story short. ajax is a big part of every project these days. every framework handle the ajax in deferent way, but non of em focused on ajax completely. all frameworks just handle a little part of ajax and in most of the time, every usability, needs deferent library. for example dealing with uploading files, restful, Bigpipe and loading JS and CSS files and etc.

How to use

using this framework is easy. just need to put this code in head's tag of your project and then our lovely bee can start making honey into the honeycomb.

<script src="javascripts/bee.min.js" type="text/javascript"></script>

Honeycomb

honeycomb is a module that handle the ajax requests.

var beeObj = bee.wizz('honeycomb')
   .request('http://domain.com/index.html')
   .method('get')
   .on('success', function (result) {
      // result 
      console.log('success', result);
    })
   .resultData('text')
   .spread();

list of the available methods:
Method Description
request(PATH:string) request's method gets the url of the destination path
method(METHOD:string) request header's method. (get, post, put, delete, head , ...)
resultData(TYPE:string) represent the type of the result that you are expect.
(text, json, xml)
data(METHOD:string, DATA:object, MERGE:bool(true)) attach data to the request.
  • method: (get, post)
  • data: data object
  • merge: merge the querystring's data and attached data to gether.
    this option is working only on get method
    (optional): default value true
files(FILES:objectArray) an array object includes File objects
[
new File(...),
new File(...),
]
noAsync() this method deactivate the async mode of your request
credential(USERNAME:string, PASSWORD:string) allows you to set credential to the request.
on(EVENT:string, CALLBACK:function) events that trigger in specific situation

on(event, callback) events list:

event description code
beforestart on('beforestart', function(){
// your codes
}) 		calls before start sending the request
success on('success', function(data){
// your codes
}) 		return the request response.
notice that the data variable return the data according to resultData(type)

resultData(type)

text: return plan text data
json: return an object
xml: return a xml object
fail on('fail', function(){
// your codes
}) 		calls when request has been failed.
Failure may have many reasons such as:
404 Error, 500 Error or ...
cancel on('cancel', function(){
// your codes
}) 		calls when request has been canceled by user from web browser.
progress on('progress', function(ProgressEvent){
// your codes
}) 		calls when data is about to upload.
generaly: it return and XMLHttpRequestProgressEvent object to the ProgressEvent which includes some data about the amount of loaded data or total amount of data.

XMLHttpRequestProgressEvent

Script

This module manage the script tags and put them to the head's tag then execute it. 

var beeObj = bee.wizz('script')
   .src('/javascripts/script.js')
   .charset('UTF-8')
   .type('text/javascript')
   .async()
   .on('beforeload', function (result) {
      // before the script tag start to fetch the uri's data 
      console.log('before start download the script', result);
    })
   .on('ready', function (result) {
      // when the script file has been downloaded and executed 
      console.log('script file is ready', result);
    })
   .on('error', function (result) {
      // when script's file does not exist or access denied 
      console.log('script downloading has been failed', result);
    })
   .spread();

list of the available methods:
Method Description
src(PATH:string) defines the script file destination path
charset(CHARSET:string) you can change the charset of the script tag. (UTF-8 is the default value)
type(TYPE:string) represent the type of the script tag that you are expect.
(text/javascript, text/vbscript or ...)
async() change the script loading mode to async.
defer() change the script loading mode to defer.
code(CODE:function) it allows you to inject your inline script into the head's tag
on(EVENT:string, CALLBACK:function) events that trigger in specific situation

on(event, callback) events list:

event description code
beforestart on('beforestart', function(){
// your codes
}) 		calls before start import the script tag into head's tag
ready on('ready', function(){
// your codes
}) 		dispatch when your script downloaded successfully and execute as well.
error on('error', function(){
// your codes
}) 		calls when request has been failed.
Failure may have many reasons such as:
404 Error, 500 Error or ...

BigPipe

By passing the time, every websites become biger than before. and this may cause of some performance issue for developers and actually websites.

Big pipe helps projects to change their focus point from whole page to every small part of each page which called pagelet instead of having deal with the whole sources in one repository.

It means that every pages that user reach can chunk to smaller parts.
for example, if you have a look on facebook, you may notice that the page divided to some parts and each part load in deferent ajax request.

If you want to load the whole data in one page at once, you can notice that you page load is really slow.

well, what do we should do???
it's really easy. you can load every single part of your page as a pagelet to your page and make sure that every pagelet has its own css and script and it will execute whenever you want.
by doing this, you can see that your page probibly has some more page requests, but your page load time is really lower than before.
